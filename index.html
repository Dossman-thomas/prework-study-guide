HTML


Hypertext Markup Language (HTML) is made up of different elements (wrapped in angled brackets, <" "> </" ">) that create the content and basic framework for all webpages.



An element is the type of content on a webpage. Common examples of elements include:

  • Text elements
  • Image elements
  • Header elements
  • List elements


Elements are also used to divide and describe content on a page. Examples of such elements include:

  • Meta-related elements
  • Content division elements
  • Semantic elements


The two most important elements for the structure of the webpage are the "head" and "body" elements.

  • The head element contains information, or metadata (unseen data about webpages that the browser uses to serve the webpage to users) about the webpage that is not visible to the user.
  • The body element contains the content that is visible to the user. All the content that we want the user to see and interact with are put inside this element.


NOTE: these two elements are examples of semantic HTML elements, or elements that describe their content and the role they play on the page. It's important to use semantic elements whenever possible, because they make your HTML easier to read for other developers and the browser.

The "head" element

The head element may contain the following elements:

  • The 'charset' element sets the character encoding.
  • the 'http-equiv' element sets an HTTP header for conent.
  • The 'viewport' element sets the default view for our webpage.
  • The 'title' element defines the title of the webpage and is the only visible element in the head element.

The "body" element

The body element may contain other semantic elements including the following:

  • The 'header' element appears at the top of the page and usually contains a navigation menu, a title heading, and/or other introductory content.
  • 'heading' elements (e.g., h1, h2, h3, etc).
  • 'section' elements that divide the contents of the webpage into easier to read segments.
  • 'paragraph' elements (p) that contain text.
  • 'ul', 'ol', and 'li' elements represent unordered lists, ordered lists, and list items respectively.
  • A 'br' element creates a line break between blocks of content.
  • The 'a' element (stands for anchor) creates links to the same webpage or other webpages you provide the URL for using the 'href' attribute
  • The link element specifies relationships between the current document and an external resource. The 'src' attribute defines the location of the file. The 'alt' attribute contains a text string that describes the appearance and functionality of the image. This is important for improving Search Engine Optimization (SEO), screen readers, and other assistive technologies.
  • The 'footer' is last and appears at the bottom of the page. Usually, the footer element contains the author, copyright, contact, sitemap, and navigation of the webpage.



An important takeaway from this section is that we can use semantic HTML elements to indicate the role of their content, as well as to give structure to the content of the webpage.



NOTE: an attribute either modifies the default functionality of an element type or provides functionality to certain element types unable to function correctly without them. Another example of important attributes are the "class" and "id" attributes. They help to name elements so they can later be targeted and used with CSS and JavaScript.


CSS


Cascading StyleSheets (CSS) is a programming language that determines how webpage content is presented to users by defining colors, aesthetics, visual layouts, and organization to specific elements of an HTML page.


There are three ways to style a webpage using CSS:

  • Inline CSS - used when the developer wants to style an element directly in the HTML file. They're useful for quick and permanent changes but are less flexible than external and internal style sheets, because each style must be edited individually whenever you make a design change.
  • Internal CSS - embeds an entire stylesheet directly in the HTML file using a "style" element in the "head" element.
  • External CSS (most common) - Preferred in most cases because it allows developers to keep their CSS rules in a seperate file, making design changes easier. When using an external style sheet, we link the file using a "link" element placed in the HTML's head section. Use <link rel=”” href=””> to link a css file to your html file (rel attribute specifies the relationship between the current and linked documents and href specifies the location of the exernal resource.)


NOTE: Separation of concerns (a widely used term in web development) refers to separating code into different files based on use and the idea that each section of code should have its own responsibility.



Selectors are used to define the element or attributes to which the CSS rules, or declarations, will apply. We use the "class" attribute as a selector to target certain elements on the page we'd like to apply our CSS rules to. This helps us to prevent from repeating lines of code for elements we want to style the same.

You can target more than one selector by seperating them with a comma in CSS (e.g., .container, .text {};).


Declarations contain two important components:

  • The CSS property we want to apply
  • The value of the property

EXAMPLE: the declaration {color: blue;} includes the property "color" and the value of the property, "blue"


It's Important to note that every HTML element is essentially a box. Each box consists of content that is contained by padding, borders, and margin. This is more commonly referred to as the CSS box model and is extremely important to understand while learning this language.


There are several properties we can assign to HTML elements to style them, including, but not limited to:

  • Padding - adds space around the content inside an element. Increasing the padding makes the entire element bigger!
  • Border - determines the size, style, and color of a line drawn around the element it's applied to.
  • Margin - adds space around the outside of an element.
  • Width and Height - defines the value of the width and the height of an element. NOTE: these are seperate properties.
  • Box-shadow - adds shadow effects around an element's frame. This allows you to describe the X and Y offsets relative to the element, the blur and spread radius, and the color of the shadow.
  • Background-color - defines the color of the background of an element.
  • Color - defines the text color of an element.
  • Text-align - allows you to define the alignment of the paragraphs in your webpage.
  • and many more!


all CSS properties and descriptions of them are easily found online using Google!
