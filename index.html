HTML


Hypertext Markup Language (HTML) is made up of different elements (wrapped in angled brackets, <" "> </" ">) that create the content and basic framework for all webpages.



An <element> is the type of content on a webpage. Common examples of elements include:

  • Text elements
  • Image elements
  • Header elements
  • List elements


Elements are also used to divide and describe content on a page. Examples of such elements include:

  • Meta-related elements
  • Content division elements
  • Semantic elements


The two most important elements for the structure of the webpage are the <head> and <body> elements.

  • The <head> element contains information, or metadata (unseen data about webpages that the browser uses to serve the webpage to users) about the webpage that is not visible to the user.
  • The <body> element contains the content that is visible to the user. All the content that we want the user to see and interact with are put inside this element.


NOTE: these two elements are examples of semantic HTML elements, or elements that describe their content and the role they play on the page. It's important to use semantic elements whenever possible, because they make your HTML easier to read for other developers and the browser.

The <head> element

The <head> element may contain the following elements:

  • The 'charset' element sets the character encoding.
  • the 'http-equiv' element sets an HTTP header for content.
  • The 'viewport' element sets the default view for our webpage.
  • The 'title' element defines the title of the webpage and is the only visible element in the head element.


The <body> element

The <body> element may contain other semantic elements including the following:

  • The <header> element appears at the top of the page and usually contains a navigation menu, a title heading, and/or other introductory content.
  • 'heading' elements (e.g., <h1>, <h2>, <h3>, etc).
  • <section> elements that divide the contents of the webpage into easier to read segments.
  • 'paragraph' elements (<p>) that contain text.
  • <ul>, <ol>, and <li> elements represent unordered lists, ordered lists, and list items respectively.
  • A <br> element creates a line break between blocks of content.
  • The <a> element (stands for anchor) creates links to the same webpage or other webpages you provide the URL for using the 'href' attribute
  • The <link> element specifies relationships between the current document and an external resource. The 'src' attribute defines the location of the file. The 'alt' attribute contains a text string that describes the appearance and functionality of the image. This is important for improving Search Engine Optimization (SEO), screen readers, and other assistive technologies.
  • The <footer> is last and appears at the bottom of the page. Usually, the footer element contains the author, copyright, contact, sitemap, and navigation of the webpage.



An important takeaway from this section is that we can use semantic HTML elements to indicate the role of their content, as well as to give structure to the content of the webpage.




CSS


Cascading StyleSheets (CSS) is a programming language that determines how webpage content is presented to users by defining colors, aesthetics, visual layouts, and organization to specific elements of an HTML page.


There are three ways to style a webpage using CSS:


  1. Inline CSS - used when the developer wants to style an element directly in the HTML file. They're useful for quick and permanent changes but are less flexible than external and internal style sheets, because each style must be edited individually whenever you make a design change.
  2. Internal CSS - embeds an entire stylesheet directly in the HTML file using a <style> element in the <head> element.
  3. External CSS (most common) - Preferred in most cases because it allows developers to keep their CSS rules in a seperate file, making design changes easier. When using an external style sheet, we link the file using a <link> element placed in the HTML's head section. Use <link rel=‚Äù‚Äù href=‚Äù‚Äù> to link a css file to your html file (rel attribute specifies the relationship between the current and linked documents and href specifies the location of the external resource.)

NOTE: Separation of concerns (a widely used term in web development) refers to separating code into different files based on use and the idea that each section of code should have its own responsibility.



  • Selectors - Are used to define the element or attributes to which the CSS rules, or declarations, will apply. We use the "class" attribute as a selector to target certain elements on the page we'd like to apply our CSS rules to. This helps us to prevent from repeating lines of code for elements we want to style the same.

    You can target more than one selector by seperating them with a comma in CSS:

    .container,
    .text,
    .button {

    //block of code;

    }
  • Declarations contain two important components:

    1. The CSS property we want to apply
    2. The value of the property

EXAMPLE: the declaration {color: blue;} includes the property "color" and the value of the property, "blue"


It's Important to note that every HTML element is essentially a box. Each box consists of content that is contained by padding, borders, and margin. This is more commonly referred to as the CSS box model and is extremely important to understand while learning this language.


There are several properties we can assign to HTML elements to style them, including, but not limited to:


  • Padding - adds space around the content inside an element. Increasing the padding makes the entire element bigger!
  • Border - determines the size, style, and color of a line drawn around the element it's applied to.
  • Margin - adds space around the outside of an element.
  • Width and Height - defines the value of the width and the height of an element. NOTE: these are seperate properties.
  • Box-shadow - adds shadow effects around an element's frame. This allows you to describe the X and Y offsets relative to the element, the blur and spread radius, and the color of the shadow.
  • Background-color - defines the color of the background of an element.
  • Color - defines the text color of an element.
  • Text-align - allows you to define the alignment of the paragraphs in your webpage.
  • and many more!


all CSS properties and descriptions of them are easily found online using Google!



Git

  • Git (Global Information Tracker) - A version control system that tracks changes in any set of computer files, usually used for coordinating work among programmers who are collaboratively developing source code during software development.
  • GitHub - A platform and cloud-based service for software development and version control using Git, allowing developers to store and manage their code. GitHub is also accessible via a downloadable desktop version.
  • Git Bash - A source control management system for Windows that allows users to type Git commands that make source code management easier through versioning and commit history.
  • Github Flow - The branch-based workflow by GitHub, made to help streamline the process of developing source code within a team. To follow GitHub flow, you need a GitHub account and a repository (a platform to contain all of your project's files and revision history.)

NOTE: You can complete all steps of GitHub flow through the GitHub web interface, the command line/Git Bash, or GitHub Desktop.



The GitHub Flow:


  1. Create a Branch Create a branch in your repository. A short, descriptive branch name enables your collaborators to see ongoing work at a glance.

    git checkout -b <new branch name>

    • the "git" command must precede each Git statement.
    • the "checkout" command is to move the working branch to a new branch.
    • It is good practice to name the branches for the feature that will be developed by them to help indicate the purpose of each branch.
  2. Make Changes On your branch, make any desired changes to the repository. Your branch is a safe place to make changes. If you make a mistake, you can revert your changes or push additional changes to fix the mistake.
  3. Commit and Push Give each commit a descriptive message to help you and future contributors understand what changes it contains. Ideally, each commit contains a complete change.

    By committing and pushing changes, your work is backed up to remote storage allowing you to access your work from any device! It also allows your collaborators to view your work, respond to questions, and make contributions.

    • add changed files in your working directory to the staging area of Git:
      • git add -A

    • Now that all of our changes have been staged, we can commit to our repository and add a simple message associated with the commit:
      • git commit -m 'add message here'

    • Next, let's make sure our current branch is up to date before opening a pull request. It is good habit to pull the base branch into your feature branch before pushing up any code:
      • git pull origin main

    • Push up all of your changes using the following command:
      • git push origin "branch name"

  4. Create a Pull Request - Ask for feedback on your changes. In addition to filling out the body of the pull request, you can add comments to specific lines of the pull request to explicitly point something out to reviewers.
  5. Address Review Comments - Reviewers can comment on the whole pull request or add comments to specific lines or files. You can continue to commit and push changes in response to the reviews and you pull request will update automatically.
  6. Merge Your Pull Request - Once your pull request is approved, you may merge your pull request. This will automatically merge your branch to the default/master/main branch. GitHub will retain the history of comments and commits in the pull request to help future collaborators understand your changes.
  7. Delete Your Branch - After merging your pull request, delete the branch. This indicates that branch's work is complete and prevents anyone from accidentally working on old branches.

OTHER USEFUL GIT COMMANDS:

pwd - (print working directory) - shows the directory you're currently working on.

ls - lists files and folders in the working directory.

cd - change directory to your chosing.

cd .. - revert to the last directory you were in.

checkout "branch name" - switches to that specified branch.

log - shows the commit history for the currently active branch.

Assigning a Variable

  • Variables - Assignable containers that store different data types. To assign a variable, begin by typing 'let' if the variable will change,'const' if it will stay the same, or 'var', followed by a name of your chosing. When naming variables, it's important to note that Javascript is case-sensitive! This means the names 'myVariable' and 'myvariable' are not the same.

    After declaring a variable, give it a value by equating (=) it to a data type. for example:

    let myVariable = 5;

NOTE: it's common practice to use camel case when naming in JavaScipt.

To give a couple examples - instead of 'myvariable' and 'mysecondvariable', use 'myVariable' and 'mySecondVariable' as your variable names.


Features and Data Types

  1. String - a sequence of characters and the most commonly used data type to store text. Signified by '<string of text>' or "<string of text>"
  2. Number - Numbers don't need ' ' or " " marks. Typing numbers directly into code works.
  3. Boolean - Used to store the values true and false in a database. They're most commonly used in databased to represent yes/no, on/off, or other related states. Boolean values have one of two possible values - Truthy (true) and Falsy (false) values.
  4. Undefined - A variable has been declared but has not yet been assigned a value.
  5. Null - Represents the intentional absence of any object value. The number variable is then empty at the moment but may change value later.
  6. Object - This can be anything. Everything in JavaScript is an object and can be stored in a variable.

Using Conditional Logic

  • Conditionals - Code structures used to test if an expression returns true or not.
  • If Statements - a way of introducing decision-making into our code. Just like its name sounds, when using the "if" statement, code will only execute if the statement meets a condition, or is true.

    Creating an "if" Statement:
    if (condition){
    // block of code to be executed if condition is truthy
    }
    When the computer encounters an "if" statement in a JavaScript file, it will check to see if the condition that we provide is truthy. If it is, then it will do the action we ask it to do between the curly brackets. If the condition isn't true or has a value that isn't truthy, then the computer will ignore whatever is in between the curly brackets and move on. Using conditional statements such as "if" statements allows us to interrupt control flow.
  • "else if" and "else" Statements - Provides an if statement with more than one condition.

    Here are a couple examples:
    if (condition){

    // block of code to be executed if condition is truthy

    } else if (condition){

    // block of code to be executed if first condition is falsy and this condition is truthy

    } else {

    // block of code to be executed if none of the conditions are truthy

    }
    let dinner = "tacos";

    if (dinner === "tacos"){

      alert("Yay, I lOVE TACOS!");

    } else {

      alert("Awww, but tacos are my favorite...");

    }

    • NOTE: we use '===', the strict equality operator, to check to see if two values are equal, and return a Boolean result, true, if the values are equal, and false, if the values are not equal.

Using Arrays and Loops

  • Arrays - A single variable used to hold a group of data that is typically related in some way. To create an array, declare a variable like we have before, but instead of assigning the variable to a single value, assign it to a group of values between square brackets [].

    let myArray = ['apple','cherry','banana'];

    To work with individual data items inside an array, we need the location of the data item in the array. Using each piece of data's index, or location number, we can access individual data inside the array.

    In JavaScipt, arrays are zero-indexed and sequential. This means that the first item in myArray above, 'apple', has an index of 0. Because the indices are sequential (increasing by 1), 'cherry' then has an index of 1, and 'banana' 2.

    As an example, we can log 'banana' in our console with the following code:

    let myArray = ['apple','cherry','banana'];

    console.log(myArray[2]);


    NOTE: A single array can store different types of data, such as strings, numbers, and so on.

  • The for Loop Uses the predictable pattern of indices to perform a task on all the items in an array by allowing a block of code to be executed over and over.

    A for loop contains the following three important statements, in order:

    1. Determine the starting point of our loop. Since we want to start at the beginning of the array, we declare a variable x and give it a value of 0 (because the first index of an array is always 0).
    2. Set the condition. As long as the number in the variable x is less than the length of the array, the loop will keep running. This will allow the loop to execute one time for each item in the array. Written out:
      x < myArray.length;
    3. Indicate the iteration Since indices follow a predictable pattern, we know the number increases by 1 each time. The code x++ is a shortcut way of writing x+1. Now every time the code loops, x will increase by 1
    Therefore, to write for loops, your code should look like this:
    for(let x = 0; x < myArray.length; x++){

    // code block to execute;

    }

Writing and Calling Functions

  • Functions - A powerful core component of many programming languages ‚Äî not just JavaScript ‚Äî that allows developers to reuse code to perform certain tasks. It's important to remember that functions don't automatically run when the page is loaded. In order for them to work, you must call them within the code.
  • Write a function - Begin by typing 'function', a function name, parentheses (to define parameters), and squiggly brackets {to encapsulate blocks of code}. Remember that even if we don't define parameters, it's important to always include the parentheses.

    Here's an example of a function to log the string, "Hello, world!" in the console:
    function helloWorld ( ) {

      console.log("Hello, world!")

    }
  • Calling the function - Now, just because the function above was created, doesn't mean it'll run the code inside. In order to use the code in the function, we must call it by typing the function name, parentheses, and semi-colon.
    helloWorld ( );

    Don't forget: since control flow dictates that Javascript files are executed from top to bottom, where you call your functions matters!
